# 프로젝트 VI - 요구사항 정의서

## 1. 개요 (Introduction)
### 1.1 문서의 목적
본 문서는 '클라우드 파일 공유 시스템 (미니 드라이브)'의 개발 범위를 명확히 하고, 사용자의 요구사항과 개발 관련 요구사항을 정의하여 스테이크홀더 간의 원활한 의사소통 및 시스템 개발의 기준으로 삼기 위해 작성되었다[cite: 7].

### 1.2 프로젝트 개요
약 200명의 사내 직원이 대용량 파일을 지연 없이 업로드/다운로드하고, 외부 지사 및 모바일 환경에서도 안전하게 협업할 수 있는 중앙 집중형 클라우드 파일 공유 시스템을 구축한다.

### 1.3 관련 문서, 용어 및 약어
* 관련 문서: 프로젝트 관리 계획서 (Project_Management_Plan.md), 시스템 품질 요소 추정서 (System_Quality_Attributes.md)
* CRD: Customer Requirement Document (고객 요구사항 정의서)

---

## 2. 요구사항
요구사항은 유일한 식별자(ID)를 가지며, 단문 형태로 명확하게 기술하고 우선순위를 부여하여 관리한다[cite: 7].

### 2.1 기능적 요구사항
소프트웨어 시스템이 반드시 수행해야 하는 핵심 기능 요구사항이다[cite: 7].

| 식별자(ID) | 그룹(분류) | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| FR-001 | 파일 관리 | 사용자는 웹 브라우저를 통해 대용량 파일을 업로드한다. | High |
| FR-002 | 파일 공유 | 사용자는 파일 공유를 위한 고유 URL 링크를 생성한다. | High |
| FR-003 | 권한 관리 | 사용자는 문서별로 읽기 또는 수정 권한을 다르게 설정한다. | High |
| FR-004 | 협업 지원 | 사용자는 동일한 문서에 대해 실시간으로 동시에 수정 작업을 진행한다. | Medium |
| FR-005 | 복구 관리 | 시스템은 사용자가 삭제한 파일을 휴지통에 보관하여 복구 기능을 제공한다. | Medium |

### 2.2 비기능적 요구사항
소프트웨어 시스템이 제공해야 하는 행위적, 성능적, 보안적 속성이다[cite: 7].

| 식별자(ID) | 그룹(분류) | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| NF-001 | 성능 | 시스템은 200명 동시 접속 시에도 검색 및 다운로드 지연 시간을 1초 이내로 유지한다. | High |
| NF-002 | 보안 | 시스템은 서버에 저장되는 모든 데이터를 암호화한다. | High |
| NF-003 | 운영 (플랫폼) | 시스템은 안드로이드 및 iOS 환경에서 전용 모바일 앱으로 동작한다. | Medium |
| NF-004 | 자원 | 시스템은 클라우드 환경에서 오토 스케일링(Auto Scaling)을 통해 가용성을 확보한다. | High |

### 2.3 인터페이스 요구사항
시스템 사용 및 연동 과정에서 발생하는 요구사항이다[cite: 7].

| 식별자(ID) | 그룹(분류) | 요구사항 상세 내용 | 우선순위 |
| :--- | :--- | :--- | :--- |
| IR-001 | 시스템 연동 | 시스템은 사내 인사 관리 시스템(HR)과 연동하여 직원 계정 정보를 자동으로 동기화한다. | Medium |
| IR-002 | 사용자 인터페이스 | 시스템은 특별한 프로그램 설치 없이 브라우저의 드래그 앤 드롭 UI를 지원한다. | Low |

## 3. 품질 요구사항
이전 프로젝트 단계에서 식별한 품질 요소를 기반으로 정의한다[cite: 7].

### 3.1 사용성 및 협업 지원성
* 시스템은 별도의 교육 없이도 사용 가능한 직관적인 인터페이스를 제공해야 하며, 이전 버전 목록 확인 기능을 통해 원활한 협업을 지원해야 한다.

### 3.2 정확성 및 데이터 무결성
* 대용량 파일 전송 및 다중 기기 동기화 과정에서 단 1바이트의 데이터 손실이나 변형도 발생하지 않아야 한다.

### 3.3 이식성 및 테스트 용이성
* 특정 하드웨어에 종속되지 않고 컨테이너 기술을 통해 배포되어야 하며, 자동화된 CI/CD 파이프라인을 통해 부하 테스트가 가능해야 한다.

## 4. 개발 요구사항
시스템 구현을 위해 요구되는 소프트웨어 및 하드웨어 인프라 환경이다[cite: 7].

* **소프트웨어 요구사항:** Java 17, Spring Boot, React.js, Git, Jira
* **하드웨어(클라우드) 요구사항:** AWS EC2(운영 서버), AWS S3(파일 저장소), AWS RDS(데이터베이스)

---

## 5. 기타 고려 사항
* 공공기관 등 외부와의 협업 가능성을 고려하여 클라우드 보안인증(CSAP) 기준을 준수하여 아키텍처를 설계한다.
