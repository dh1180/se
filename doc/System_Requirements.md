# 시스템 요구사항 정의서 (v0.1)
**프로젝트명:** 클라우드 파일 공유 시스템 (미니 드라이브)

## 1. 시스템 목표
- 조직 내 파일을 중앙에서 관리
- 특별한 프로그램 설치 없이 웹 브라우저를 통한 접근 지원
- 보안을 유지하며 외부 사용자와 파일 공유 가능

## 2. 핵심 기능 명세
### 2.1. 파일 저장 및 관리
- 문서, 이미지, PDF, 압축 파일, 영상 등 다양한 일반 업무 파일 저장
- 파일 이름, 업로드 날짜, 파일 크기 정보 함께 저장

### 2.2. 폴더 체계화
- 여러 단계로 구성 가능한 폴더 생성
- 폴더 생성, 이름 변경, 삭제 및 파일 이동 기능
- 삭제된 파일의 일정 기간 복구 기능

### 2.3. 파일 공유 및 권한
- 사용자 간 파일 공유 시 보기, 수정, 댓글 작성 권한 개별 설정
- 외부 협력 업체를 위한 만료 기간 설정 가능한 공유 링크 제공

### 2.4. 검색 및 버전 관리
- 파일 이름, 특정 날짜 이후, 특정 사용자, 특정 파일 유형 기준 제한 검색
- 파일 수정 시 이전 버전 보관, 목록 확인, 다운로드 및 복원 지원

### 2.5. 보안 및 계정 관리
- 이메일과 비밀번호를 이용한 로그인
- 권한 기반 접근 제어 및 데이터 전송 시 암호화
